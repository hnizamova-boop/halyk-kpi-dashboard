## 2. Контекст и бизнес-цели

### 2.1 Предпосылки

Управляющая компания Группы Халык осуществляет надзор за пятью дочерними структурами, работающими в сегментах инвестиционного банкинга, страхования жизни, общего страхования, лизинга и брокерских услуг. Отсутствие единой системы мониторинга приводит к:

- **Разрозненному сбору данных** — KPI собираются из разных источников (Excel, почта, устные отчёты)
- **Задержкам в эскалации** — просроченные задачи выявляются постфактум, а не в режиме реального времени
- **Низкой прозрачности** — руководство группы не имеет единой точки входа для оценки состояния дочек
- **Потерям при консолидации** — ручная сводка данных занимает значительное время и содержит ошибки

### 2.2 Цели продукта

- Обеспечить единый цифровой инструмент контроля KPI и задач по всем 5 дочерним компаниям
- Сократить время подготовки управленческой отчётности с дней до минут
- Обеспечить проактивную сигнализацию по просроченным задачам и отклонениям KPI
- Предоставить ИИ-аналитику прямо в интерфейсе без дополнительных инструментов

### 2.3 Целевые пользователи

| Роль | Основная потребность | Ключевые функции |
|---|---|---|
| Управляющая компания (CFO/CEO) | Контроль группы в целом | Сводный дашборд, ИИ-анализ |
| Финансовый контролёр группы | Сравнение план/факт KPI | KPI-таблица, выгрузка в Excel |
| Руководители дочек | Статус своих задач и показателей | Фильтр по компании, трекер задач |
| Ответственные исполнители | Обновление прогресса задач | Редактирование задач |

## 3. Функциональные требования

### 3.1 Модуль 1 — Трекер задач

#### 3.1.1 Описание

Интерактивный список поручений по всем дочерним компаниям с цветовой индикацией статуса и сроков.

#### 3.1.2 Функции

- Создание, редактирование и удаление задач
- Поля задачи: название, дочерняя компания, ответственный, дата постановки, срок исполнения, прогресс (%), приоритет, комментарий
- Автоматический расчёт статуса: Просрочено / Срок близко (≤3 дня) / В работе / Не начато / Выполнено
- Цветная полоса-индикатор слева от задачи: красная / янтарная / синяя / серая / зелёная
- Фильтрация по статусу (чипы) и по дочерней компании (выпадающий список)
- Мгновенная отметка выполнения через чекбокс
- Автосортировка: просроченные — наверху
- Метрики панели: всего / просрочено / срок близко / % выполнено

#### 3.1.3 Правила статусов

| Статус | Цвет | Условие | Приоритет сортировки |
|---|---|---|---|
| Просрочено | 🔴 Красный | Срок < сегодня, не выполнено | 1 |
| Срок близко | 🟡 Янтарный | Срок через 0–3 дня | 2 |
| В работе | 🔵 Синий | Прогресс > 0%, не выполнено | 3 |
| Не начато | ⚪ Серый | Прогресс = 0% | 4 |
| Выполнено | 🟢 Зелёный | Отмечено выполненным | 5 |

---

### 3.2 Модуль 2 — KPI-дашборд

#### 3.2.1 Вкладка «Обзор группы»

- 4 метрики холдинга: активы группы, чистая прибыль, ROE, достаточность капитала
- Карточки 5 дочерних компаний: прибыль, ROE, прогресс-бар выполнения KPI с цветным индикатором
- Столбчатая диаграмма: прибыль по компаниям (млрд ₸)
- Диаграмма: % выполнения плана KPI — с линией плана (100%)

#### 3.2.2 Вкладка «Ежемесячные показатели»

- Линейный мультиграфик: динамика чистой прибыли по всем дочкам за 12 месяцев
- Отдельные графики: активы и NPL (для банковских и лизинговых структур)
- Навигация по периоду (кнопки «назад / вперёд»)
- Легенда с цветами каждой компании

#### 3.2.3 Вкладка «KPI план/факт»

- Таблица: показатели по строкам, компании по столбцам (план | факт | % выполнения)
- 9 KPI-строк: чистая прибыль, ROE, активы, кредитный портфель, NPL, GWP, клиентская база, кост/инком, ROA
- Цветовая легенда: 🟢 ≥100% / 🟡 85–99% / 🔴 <85%
- Фильтр по конкретной дочерней компании

#### 3.2.4 ИИ-аналитика (интегрирована во все вкладки)

- Кнопка «Анализ ИИ» на каждой вкладке
- Автоматическая сборка контекстного промта из текущих данных
- Запрос к Anthropic Claude API (`claude-sonnet-4-6`)
- Вывод: оценка рисков, топ-рекомендации, комментарий по динамике или просрочкам

---

### 3.3 Модуль 3 — Excel-шаблон (`Halyk_KPI_Template.xlsx`)

Структурированный файл для ввода реальных данных с автоматическим расчётом и последующей загрузкой в дашборд.

#### 3.3.1 Листы файла

| Лист | Содержание | Тип данных |
|---|---|---|
| Инструкция | Цветовая легенда, правила заполнения | Справочный |
| Ежемес. показатели | 7 KPI × 12 месяцев × 5 компаний | Ввод (синие ячейки) |
| KPI план-факт | 9 KPI × план/факт/% × 5 компаний | Ввод + формулы |
| Задачи | 30 строк с выпадающими списками | Ввод |
| Сводка группы | Автосводка из KPI-листа | Только формулы |

#### 3.3.2 Стандарты оформления

- 🔵 **Синий текст** (`#0000FF`) — вводимые значения
- ⚫ **Чёрный текст** — формулы (не редактировать)
- 🟡 **Жёлтый фон** (`#FFFDE7`) — обязательные к заполнению ячейки
- Нулевые ошибки (`#REF!`, `#DIV/0!`) — проверено скриптом `recalc.py`
- Валидация: выпадающие списки для статуса задачи, приоритета, компании

## 5. Нефункциональные требования

### 5.1 Производительность

- Дашборд загружается и рендерится менее чем за 2 секунды при типичном объёме данных
- ИИ-анализ возвращает ответ менее чем за 10 секунд (зависит от Anthropic API)
- Excel-шаблон пересчитывает формулы менее чем за 3 секунды

### 5.2 Совместимость

- Веб-дашборд: Chrome 110+, Edge 110+, Firefox 115+, Safari 16+
- Excel-шаблон: Microsoft Excel 2019+, Excel 365, LibreOffice Calc 7+
- Адаптивность: корректное отображение на экранах от 1280px шириной

### 5.3 Безопасность и конфиденциальность

- Финансовые данные не передаются в Anthropic API — в ИИ-промт включается только агрегированная статистика
- Доступ к файлу Excel регулируется правами на уровне корпоративной файловой системы
- Дашборд не хранит данные в браузере (`localStorage` не используется)

### 5.4 Поддерживаемость

- Все дочерние компании настроены как массив — замена или добавление компании не требует переработки логики
- KPI-показатели добавляются через единый массив конфигурации

## 6. Архитектура решения

### 6.1 Компоненты

| Компонент | Технология | Назначение |
|---|---|---|
| Трекер задач | HTML/JS (React-виджет) | Управление поручениями в реальном времени |
| KPI-дашборд | HTML/JS + Chart.js 4.4 | Визуализация показателей по вкладкам |
| ИИ-аналитика | Anthropic API (`claude-sonnet-4-6`) | Генерация управленческих рекомендаций |
| Excel-шаблон | openpyxl + LibreOffice recalc | Ввод и хранение реальных данных |
| Документация | PRD.md / PRD.docx | Спецификация и передача в разработку |

### 6.2 Поток данных

```
Пользователь заполняет Excel-шаблон (синие ячейки)
        ↓
Загружает файл в Claude → данные парсятся
        ↓
Дашборд визуализирует данные, статусы считаются в браузере
        ↓
Кнопка «Анализ ИИ» → контекстный промт → Anthropic API
        ↓
Ответ отображается в чате рядом с виджетом
```
